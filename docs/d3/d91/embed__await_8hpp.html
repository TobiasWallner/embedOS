Replaces co_await suspension points with a safe alternative.

co_await can sadly not be overloaded to check if an awaited object coforms to certain rules. In order for an awaitable ot work together with embedOS's schedulers they have to derive from embed::AwaitableNode, which performs all the necessary registrations and signaling from the coroutines leaf execution to the scheduler.

embed_await() will check, if an awaitable conforms to the embedOS eco-system.

  • If yes, then nothing happens and the embed:: awaitable is forwarded to co_await.
  • If not, then the awaitable will be wrapped to add the embed::AwaitableNode
